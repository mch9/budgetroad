## Decisions & Constraints
- **Analytics 경로 피벗**: 자체 Event DB + /admin/dashboard → GA4 + Looker Studio. 자체 DB는 Phase 2(공유 링크)/Phase 3(마이페이지) 사용자 기능 맥락에 자연 합류. 16개 KPI 중 14~15개는 GA4로 관측 가능, 코호트 KPI는 BigQuery SQL 경로로 이관.
- **"데이터 없는 조합"은 UI에서 차단**: 강남+전통혼례 같은 `VENUE_PRICES[...] === 0` sentinel 조합을 `0`/평균치로 채우지 않고 OptionCard를 disabled 렌더링 + region 변경 시 자동 리셋. 통계 기반 서비스 신뢰성 원칙.
- **스드메 시장 조정은 상수로**: `SDM_MARKET_MULTIPLIER = 1.5`를 `calculateBudget`에 곱해 원본 조사 데이터(`STUDIO_PRICES`·`MAKEUP_PRICES`·`DRESS_PRICES`) 보존. 사용자 옵션화 시 상수 → prop 교체만 하면 됨. 동시에 STUDIO/MAKEUP 가격이 서로 뒤바뀐 잠복 버그(같은 등급 선택 시 총액 우연히 일치)도 수정.
- **리브랜드 2단 진행**: 웨딩버젯 → 버젯로드(도메인 이전 포함) → 버짓로드 표기 교정. Vercel에서 프로젝트명만 바꿔도 `.vercel.app` 도메인은 자동 갱신되지 않아 Domains 패널에서 수동 교체 필요.
- **CI 이슈 3건 순차 해결**: `useRef(Date.now())` → `useState` lazy init(`react-hooks/purity`), sessionStorage 복원 effect는 SSR hydration 회피 위해 block-level `eslint-disable`, notify-complete 잡은 `'skipped'` 처리 불가한 `action-slack` 제약에 맞춰 삼항식 폴백 + PR 브랜치 skip.
- **GA4 수집 구조 확립**: 브라우저 → GA 직접 송신(자체 서버 경유 X), `/collect` 204가 정상, `visitor_id`·`is_returning`·`region`·`venue_type` 등 커스텀 측정기준은 등록 시점 이후 데이터만 리포트에서 쓸 수 있어 **첫 배포 직전 수동 등록이 바른 순서**.
- **Squash-merge-only repo 제약**: 여러 fix 커밋이 단일 squash로 합쳐지므로 PR 단위의 논리 분해가 곧 기록 단위.
