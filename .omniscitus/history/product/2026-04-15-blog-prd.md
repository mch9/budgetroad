# 블로그 PRD

## Summary
버짓로드 블로그 기능 상세 PRD. 2026-04-15에 /prd-collab Step 0~8 완료(사례 큐레이션 전제). 2026-04-18에 **Content Strategy(Step 5-0) 신설** — 외부 사례 큐레이션 → **자체 생산 3축(guide/data/frame) 전략으로 전제 전환** + 전제 변경에 맞춰 Step 1-3·태그 체계·필터·제목까지 전면 정합화. Step 9(와이어프레임)는 새 디자인 시스템 확정 후 진행 예정.

## Context
- **Background**: 시뮬레이션 결과의 광범위한 예산 범위(3,000~5,000만 원)를 보고 사용자가 막막함을 느껴 외부로 이탈하는 문제. One-Pager PRD 초안이 모호하여 /vague로 정제 후 /prd-split → /prd-collab 진행
- **Requirements**: 자체 블로그 시스템(/blog, /blog/{slug}), Markdown+DB 태그 CMS, 예산 태그(전체 예산 금액 or 항목별), 항목 태그, 결과 하단 추천 카드(경험 #5 범위), 저장은 로그인 필수
- **Decisions**: feature hierarchy = blog; 내부명 = blog; URL = /blog/{slug}; CMS = Markdown+DB 태그; 초기 콘텐츠 3개 이하로 시작, 없으면 섹션 숨김. **(2026-04-18 업데이트)** 컨텐츠 전제 전환 — 외부 사례 큐레이션 금지, **자체 생산 3축(guide/data/frame)** 채택; 태그 체계 = 예산 금액 태그 폐기 → 컨텐츠 축 태그 1개 필수 + 항목 태그; 매칭 = 선택 항목(지역·예식장·스드메 등) 기반 항목 태그 매칭; 단계별 전략 = 초반 축 1(가이드 5편) → 중반 축 2 + 서비스 데이터 연결 → 장기 축 3 + 유저 기반 사례; 초기 10편 믹스 5(guide)+3(data)+2(frame)
- **Constraints**: Step 9(와이어프레임)에서 피그마 전체 리디자인과 합류해야 함 — 새 디자인 시스템 확정 후 와이어프레임 진행 필요

### 2026-04-18
**Focus**: Content Strategy(Step 5-0) 신설 + PRD 전제 정합화
- 외부 사례 큐레이션(네이버 블로그·카페·유튜브) 접근법 검토 — 원본 링크는 법적으로 합법이나 **원저작자 정서적 리스크 + 장기 운영 부담**, 업체/상품 비교 리뷰는 정보 비대칭 + 분쟁 리스크로 제외
- **자체 생산 컨텐츠 3축** 정의:
  - 축 1 `guide` — 가이드·FAQ (AI + 도메인 지식 + 편집, SEO 강점)
  - 축 2 `data` — 데이터 해설 (한국소비자원·통계청·자체 통계, 차별성 최고)
  - 축 3 `frame` — 프레임·의사결정 (AI 강점, 공유/북마크 강점)
- 단계별 전략 확정: 초반 축 1 집중(3~5편) → 중반 축 2 + 서비스 데이터 연결 → 장기 축 3 확대 + 유저 기반 사례
- 초기 10편 믹스: 5(guide) + 3(data) + 2(frame)
- blog-v0.md 8곳 수정 — 전제(사례→가이드·데이터) 변경에 따라 Step 1(User Goal), Step 2(How This Helps), Step 3(Emotional Journey), Step 5 #2(예산 금액 태그→컨텐츠 축 태그), Step 5 #8(예산 근접 매칭→항목 태그 매칭), Step 8(필터), 제목("사례 가이드 블로그"→"버짓로드 블로그"), Last Updated 모두 정합화 + Step 5-0 신설

**Learned**: PRD 전제가 바뀔 때 **새 섹션만 추가하면 기존 진술과 충돌**해 후속 작업자(와이어프레임/UX Writing)가 어느 버전을 따라야 할지 혼란. 반드시 기존 모든 섹션을 새 전제에 맞춰 동기화해야 PRD가 **단일 진실원(single source of truth)** 역할 유지. "사례 큐레이션 vs 자체 생산" 판단은 저작권 이슈보다 **정서적·운영 리스크 + 경쟁 해자** 관점이 핵심 — 자체 생산은 카피 불가능한 자산이 됨.
