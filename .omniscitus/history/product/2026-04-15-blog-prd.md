# 사례 가이드 블로그 PRD

## Summary
결과 페이지의 숫자 간극을 실사례로 메워주는 자체 블로그(blog) 기능의 상세 PRD 작성. Step 0~8까지 완료, Step 9(와이어프레임)부터 진행 예정.

## Context
- **Background**: 시뮬레이션 결과의 광범위한 예산 범위(3,000~5,000만 원)를 보고 사용자가 막막함을 느껴 외부로 이탈하는 문제. One-Pager PRD 초안이 모호하여 /vague로 정제 후 /prd-split → /prd-collab 진행
- **Requirements**: 자체 블로그 시스템(/blog, /blog/{slug}), Markdown+DB 태그 CMS, 예산 태그(전체 예산 금액 or 항목별), 항목 태그, 결과 하단 추천 카드(경험 #5 범위), 저장은 로그인 필수
- **Decisions**: feature hierarchy = blog; 내부명 = blog; URL = /blog/{slug}; CMS = Markdown+DB 태그; 매칭 = 총 예산 구간 기본 → 글 많으면 항목별 근접 매칭; 초기 콘텐츠 3개 이하로 시작, 없으면 섹션 숨김
- **Constraints**: Step 9(와이어프레임)에서 피그마 전체 리디자인과 합류해야 함 — 새 디자인 시스템 확정 후 와이어프레임 진행 필요
