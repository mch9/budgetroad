    # 버짓로드 결과 페이지 계산 알고리즘 (v2 · 식장 추천 포함)
    
    온보딩 응답 → 유형 분류 → 추천 식장 → 예산 계산 → 정합성 진단 → 동적 조언 → 3탭 출력
    
    ---
    
    ## 파이프라인 개요
    
    ```
    STAGE 1  유형 점수 산출      (유형결정 6문항 → 축 A·B)
    STAGE 2  사분면 분류         (축 점수 → 4유형 / 미결정)
    STAGE 3  변수 세팅           (Modifier·태그 → 지역·하객·예산·베이스·토글)
    STAGE 4  추천 식장 형태      (유형 + 하객 + 예산 + 태그 + 양가)   ★ 신규
    STAGE 5  예산 계산           (식대 공식 + 스드메 + 예식장 + 기타)
    STAGE 6  정합성 진단         (유형 밴드 vs 실제 core → WARN/OVER/UNDER/FIT)
    STAGE 7  동적 조언 생성      (세이브 + 투자)
    STAGE 8  3탭 출력            (종합설계서 / 항목별내역 / 추가금케어)
    ```
    
    ---
    
    ## STAGE 1 — 유형 점수 산출
    
    유형결정 6문항(Q1·Q2·Q3·Q4·Q7·Q8)만 점수에 반영.
    
    ```
    axisA = 0 ; axisB = 0
    for q in [Q1,Q2,Q3,Q4,Q7,Q8]:
        opt = 사용자_선택(q)
        axisA += opt.a ; axisB += opt.b
    ```
    
    ### 선택지 가중치 표
    
    | 문항 | A | B | C | D |
    | --- | --- | --- | --- | --- |
    | Q1 (100만 추가) | b+2 | b−2 | a+1·b+1 | a−1·b−2 |
    | Q2 (더 아쉬운 상황) | b−2 | b+2 | — | — |
    | Q3 (식장 첫 인상) | b+2 | b−2 | — | — |
    | Q4 (20% 삭감) | a−1·b−1 | a+1·b+1 | a−2 | a−2 |
    | Q7 (솔깃한 옵션) | a+1·b+1 | a+1·b−1 | a+2·b+1 | a−2 |
    | Q8 (후보 중 첫눈에 끌리는 식장) | a+2·b+1 | a−1 | a+1·b−1 | a−2 |
    - 축 A = 격식·이벤트(+) ↔ 실용·최소화(−) / 축 B = 하객·공적(+) ↔ 개인·경험(−)
    - ※ Q2는 선택지 내용 기준으로 A=경험(b−2), B=하객(b+2)로 채점
    
    ---
    
    ## STAGE 2 — 사분면 분류
    
    ```
    if |axisA| <= 1 and |axisB| <= 1:  type = 탐색미결정
    elif axisA >= 0 and axisB >= 0:    type = 전통격식
    elif axisA <  0 and axisB >= 0:    type = 표준실용
    elif axisA >= 0 and axisB <  0:    type = 경험연출
    else:                              type = 본질미니멀
    ```

    ## STAGE 3 — 변수 세팅 (Modifier · 태그)
    
    ### M5 → 지역 프로파일 (DB 실측)
    
    | 지역 | 스드메계수 | 보증인원 | 1인당식대 | 대관비용 | 기본식대총 | 생화꽃장식 |
    | --- | --- | --- | --- | --- | --- | --- |
    | 서울 | ×1.09 | 200 | 8만 | 590만 | 1,540만 | 550만 |
    | 수도권 | ×1.01 | 200 | 7만 | 550만 | 1,450만 | 450만 |
    | 광역시 | ×1.00 | 200 | 5만 | 280만 | 940만 | 150만 |
    | 지방 | ×0.95 | 200 | 5만 | 250만 | 900만 | 100만 |
    
    ### 그 외 변수
    
    - M2 → guests (하객 수) / M1 → budgetTarget (목표 예산) / M3 → yangga (양가 0·1·2)
    - 태그: T2(드레스 지정 반응) · T5(신혼집 우선순위) · T7(하객 만족 포인트)
    
    ### 유형별 베이스 선택값
    
    | 유형 | 드레스 계약 | 메이크업 등급 | 본식 촬영 |
    | --- | --- | --- | --- |
    | 전통격식 | 본식+촬영(159) | 원장(78) | 예식장 연계(80) |
    | 표준실용 | 본식+촬영(159) | 부원장(69) | 예식장 연계(80) |
    | 경험연출 | 본식+촬영(159) | 원장(78) | 외부 전문(≈150, 추정) |
    | 본질미니멀 | 본식만(145) | 실장(63) | 예식장 연계(80) |
    
    ### 추가금 토글 디폴트
    
    ```
    toggle[k] = (k in 유형별_기본ON_목록[type])
    if yangga == 2: toggle[폐백음식,폐백수모,주례,혼주메이크업,한복대여] = ON
    if T7 == 무드집착: toggle[생화꽃장식,플라워샤워,포토테이블] = ON
    ```

    ## STAGE 4 — 추천 식장 형태 ★ 신규
    
    유형이 기본 형태를 정하고, 하객·예산이 규모·등급을, 태그·양가가 방향을 보정.
    
    ### (1) 유형 + 하객/예산 → 기본 형태
    
    ```
    전통격식:
        예산 >= 5500 and guests >= 300 → 5성급 호텔
        예산 >= 4250                   → 호텔·컨벤션 프리미엄 홀
        else                           → 컨벤션홀·격식형 웨딩홀
    표준실용:
        guests >= 300 → 대형 일반 웨딩홀
        guests >= 150 → 일반 웨딩홀
        else          → 중소형 웨딩홀
    경험연출:
        guests < 100  → 하우스·특색 소규모 베뉴
        guests <= 200 → 하우스 웨딩·가든
        else          → 컨셉형 대형 베뉴·가든홀
    본질미니멀:
        guests < 50   → 레스토랑·스몰 베뉴
        guests <= 100 → 하우스 웨딩·스몰홀
        else          → 실속형 소규모 웨딩홀
    탐색미결정:
        형태 추천 보류 (우선순위 먼저 안내)
    ```
    
    ### (2) 태그 · 양가 → 방향 보정 (이유로 노출)
    
    | 신호 | 보정 메시지 |
    | --- | --- |
    | T7 무드집착 | 분위기·조명 중시 → 공간이 예쁜 곳 우선 |
    | T7 자유추구 | 형식보다 우리다움 → 하우스·레스토랑 등 비전통 공간 적극 고려 |
    | T7 호스트 | 하객 만족 중시 → 음식·접근성 좋은 공간 우선 |
    | T5 생활우선 | 신혼집 우선 → 대관·보증 부담 적은 실속형 공간 |
    | T2 최소비용 | 추가금 최소화 → 올인원·패키지 명확한 곳 |
    | M3 양가 압력 高 | 일반 웨딩홀·호텔 선호 가능 → 형태 조율 필요 경고 |
    
    ### (3) 출력
    
    추천 형태 + 대안 + 이유(유형·하객·태그·양가 신호 라벨) → 종합 설계서 상단에 노출.
    
    ⚠️ 형태별 가격 데이터는 미보유 → 추천 형태에 맞는 정확한 예산은 STAGE 5에서 일반 웨딩홀 프로파일로 근사. 식장 형태별 DB 확보 시 보강.

    ## STAGE 5 — 예산 계산
    
    ### (1) 식대 — 보증인원 방식 (나)
    
    ```
    if guests >= 보증인원:  식대 = guests × 1인당식대 ;  대관 = 0
    else:                   식대 = 기본식대총금액      ;  대관 = 대관비용
    ```
    
    ### (2) 카테고리별 합산
    
    ```
    스튜디오 = (139 + ON스튜디오옵션) × 지역계수
    드레스   = (드레스베이스 + ON드레스옵션) × 지역계수
    메이크업 = (메이크업등급 + ON메이크업옵션) × 지역계수
    예식장   = 식대 + 대관 + 기본장식(0) + 본식촬영 + ON연출·진행항목 (생화는 지역단가)
    기타     = 예물 + 혼수 + 신혼여행 (유형별 추정)
    core     = 스드메 + 예식장
    total    = core + 기타
    ```
    
    ### (3) 기타 추정값 (DB 입력 전)
    
    | 항목 | 전통격식 | 표준실용 | 경험연출 | 본질미니멀 |
    | --- | --- | --- | --- | --- |
    | 예물/예단 | 500 | 350 | 250 | 150 |
    | 혼수 | 600 | 600 | 600 | 600 |
    | 신혼여행 | 400 | 400 | 450 | 300 |

    ## STAGE 6 — 정합성 진단
    
    "말한 유형"과 "나온 숫자(core)"의 정합성 진단.
    
    ### 유형별 core 밴드 (스드메 + 예식장)
    
    | 유형 | 하한 | 상한 |
    | --- | --- | --- |
    | 본질미니멀 | 1,000 | 2,200 |
    | 표준실용 | 2,200 | 3,400 |
    | 경험연출 | 2,700 | 4,100 |
    | 전통격식 | 3,200 | 5,000 |
    
    ### 분기 로직
    
    ```
    band = 밴드[type]
    if type == 본질미니멀 and guests < 보증인원:   status = WARN   # 보증인원 미스매치 → STAGE4 추천 스몰 베뉴로 유도
    elif core > band.상한:                          status = OVER   # 식대 / ON된 OFF-TYPE 항목 지목
    elif core < band.하한:                          status = UNDER  # 투자 권유
    else:                                           status = FIT
    ```
    
    ### OFF-TYPE 항목 (유형과 충돌)
    
    | 유형 | OFF-TYPE |
    | --- | --- |
    | 본질미니멀 | 드레스지정, 가봉스냅, 퍼스트웨어, 서브스냅, 원본구매, 2부드레스, 생화꽃장식 |
    | 표준실용 | 퍼스트웨어, 가봉스냅, 생화꽃장식 |
    | 경험연출 | (없음) |
    | 전통격식 | 퍼스트웨어 |

    ## STAGE 7 — 동적 조언 생성
    
    ### 세이브 (status별)
    
    ```
    if WARN: "추천한 스몰 베뉴로 변경" + 식대·대관 절감액
    for k in OFF-TYPE[type]: if ON → "{k} 해제 (−단가)"
    if 식대 > band.상한 × 0.45 and not WARN: "하객 규모 조정"
    if 비어있음: "현재 큰 낭비 없음"
    ```
    
    ### 투자 (유형 가치 기반)
    
    - 본질미니멀: 신혼집·자산 / 올인원 패키지
    - 표준실용: 핵심 패키지 방어 / 접근성
    - 경험연출: 스드메·본식 촬영 / 우선순위 3가지
    - 전통격식: 식사 퀄리티 / 혼주·폐백 격식

    ## STAGE 8 — 3탭 출력
    
    | 탭 | 내용 |
    | --- | --- |
    | 종합 설계서 | 유형, **추천 식장 형태(STAGE4)**, 예산 총합, 정합성 진단, 예산 이유, 동적 세이브/투자, 양가 경고, 준비 TOP 3 |
    | 항목별 내역 | 식대 산출 근거, 카테고리별 금액, 지역계수, 총합 |
    | 추가금 케어 | 유형별 토글 디폴트, 실시간 재계산(진단·총액 동시 갱신), 위험 항목 경고 |

    ## 데이터 처리 / 미해결 과제
    
    | 항목 | 상태 |
    | --- | --- |
    | 스드메·예식장 단가 | DB 실측 적용 완료 |
    | 식대 / 보증인원 / 대관 / 생화 | DB 지역별 실측 적용 완료 |
    | 스튜디오 선택 항목 단가 | DB 이상(3만 균일) → 추정 사용, **리서치팀 재확인 필요** |
    | 식장 형태별 가격 (호텔/하우스/스몰베뉴) | **미보유 → 형태 추천은 가능하나 정확 예산 불가, 데이터 요청 필요** |
    | 예물·혼수·신혼여행 | DB 없음 → 유형별 추정, **데이터 확보 시 교체** |
