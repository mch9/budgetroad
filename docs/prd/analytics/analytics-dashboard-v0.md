# 운영 분석 대시보드 — One-Pager PRD

> 작성일: 2026-04-16

## 목적

현재 사용자 행동 데이터가 GA4에만 존재하여, 핵심 퍼널 지표를 확인하려면 GA4에 접속해 커스텀 보고서를 만들어야 한다. 자체 DB에 이벤트를 쌓고, 구글 스프레드시트로 빠르게 시각화하여 운영자(본인 및 팀원)가 서비스 상태를 즉시 파악할 수 있도록 한다.

## 핵심 요구사항

### 전환 지표
1. P(First Action | Entered) — 진입 후 첫 액션 발생률
2. P(Result Viewed | Entered) — 진입 → 결과 확인 전환율
3. P(Result Viewed | First Action) — 입력 시작 → 결과 확인 전환율

### 리텐션/재진입
4. 7일 내 재진입률
5. P(Revisited | Result Viewed)
6. P(Revisited | Intent Created)

### 공유/저장 의도
7. P(Intent Created | Result Viewed) — 결과 공유/저장률
8. 공유 버튼 클릭 수

### 행동 패턴
9. 스텝 간 이전 버튼 클릭 확률 + 이전 후 이탈률
10. 각 페이지 평균 체류시간
11. 스크롤 이벤트 발생 수
12. 한 사용자 당 결과 생성 개수

### 효율
13. Entered → First Action 평균 시간
14. First Action → Result Viewed 평균 시간

### 분포
15. 일별/주별 방문자 수 (신규 vs 재방문)
16. 선택 분포 — 지역/예식유형/등급별 사용자 선택 비율

## 성공 지표

- GA 접속 없이 핵심 KPI 16개를 30초 내 확인 가능
- 퍼널 이탈 구간을 즉시 식별하여 개선 우선순위 결정 가능
- 일별 추이 변화를 감지하여 배포 후 영향 평가 가능

## 기술 방향

- **데이터**: Neon Postgres에 이벤트 테이블 추가 (Prisma 스키마 확장)
- **수집**: 프론트엔드 trackEvent()에서 자체 API로 이벤트 전송 (GA 병행 유지)
- **화면**: /admin/dashboard 경로에 관리자 페이지, recharts로 고정 차트 시각화, 기간 필터 (오늘/7일/30일)
- **연동**: 기존 localStorage visitor_id를 DB 이벤트에도 포함
- **인증**: 환경변수 기반 비밀번호 인증 (MVP)
