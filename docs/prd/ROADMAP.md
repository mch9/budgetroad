# Experience & Operations Roadmap

## 프로덕트 비전
결혼 준비 중인 사용자가 결혼 유형과 조건을 선택하면, 통계 기반 평균값으로 예산 초안을 자동 생성해주는 웹앱.
"구상 상태"에서 "초안 작성 시작 상태"로의 전이를 돕는다.
시뮬레이션 결과의 숫자 간극을 실사례 콘텐츠로 메워주어 외부 이탈을 방지하고 저장/공유 전환을 유도한다.

## 제품 구조
```
제품 = [사용자 경험 레이어]  +  [운영/도구 레이어]
        (user-facing)          (operator-facing)
```
사용자 경험은 `/prd-collab` 12단계로, 운영/도구는 `/prd-onepager` 5질문으로 각각 기획 깊이를 맞춰 작성한다.

## 사용자 경험 레이어

### 🟢 MVP
| # | 경험명 | 설명 | PRD 상태 |
|---|--------|------|----------|
| 1 | 예산 초안을 받아보는 경험 | 결혼 유형 선택 → 통계 기반 예산 자동 생성 → 테이블+차트 결과 확인 | ✅ 완료 |

### 이후 방향
MVP 이후 경험 로드맵은 **팀 합의 전이라 미확정**. 결정 시 `/prd-collab`으로 신규 작성.

## 운영/도구 레이어
운영자(본인·팀원)가 서비스를 관찰·관리하기 위한 내부 도구. 사용자 UX가 없어 `/prd-onepager`로 가볍게 기획한다.
