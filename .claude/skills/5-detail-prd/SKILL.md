# 5. Detail PRD

1번에서 정한 프로젝트 방향을 상세 PRD(Product Requirements Document)로 발전시키는 단계입니다.

## When to Use

- 사용자가 `/5-detail-prd` 을 입력했을 때

## Instructions

### Step 1: 이 단계가 뭔지 설명하기

아래 내용을 출력하세요:

---

## Step 5: 상세 기획서 (PRD) 작성

```
1번에서 만든 설계도            상세 기획서 (PRD)
┌──────────────────┐       ┌──────────────────────────┐
│ 핵심기능: 주문     │       │ 주문 기능:                │
│        결제      │       │  - 메뉴 선택 → 장바구니    │
│        알림      │  ──→   │  - 수량 변경 가능          │
│                  │  상세화 │  - 최소 주문금액 10,000원  │
│  (뼈대만 있음)     │       │  - 결제 후 주문번호 발급   │
└──────────────────┘       │  - 예상 배달시간 표시      │
                           └──────────────────────────┘
```

**쉽게 말하면:**
1번에서 "이런 집을 짓겠다"는 큰 그림을 그렸다면,
이번엔 "거실은 몇 평, 창문은 어디에, 콘센트는 몇 개" 같은
**세부 사항**을 정하는 거예요.

이게 있어야 Claude가 "아, 이 버튼을 누르면 이렇게 동작해야 하는구나"
하고 정확하게 코드를 짤 수 있어요.

---

### Step 2: 이해 여부 확인

AskUserQuestion 도구를 사용하여 질문하세요:

- question: "위 설명을 이해하셨나요?"
- header: "이해 확인"
- options:
  - label: "이해했어요" / description: "다음 단계로 넘어갑니다"
  - label: "더 설명해주세요" / description: "궁금한 점을 직접 입력해주세요"

### Step 3: CLAUDE.md 읽기

프로젝트의 CLAUDE.md를 읽어서 1번에서 정의한 핵심 기능 목록을 파악합니다.

### Step 4: /prd-split 호출

**`/prd-split` skill을 호출**하여 CLAUDE.md의 프로덕트 비전을 경험 단위로 분해하고,
각 경험마다 `/prd-collab`의 12단계 워크플로우로 상세 PRD를 작성합니다.

> `/prd-split`이 내부적으로 `/prd-collab`을 호출하므로, 이 단계에서는 `/prd-split`만 실행하면 됩니다.

### Step 5: CLAUDE.md 업데이트

PRD 작성이 완료되면 핵심 사항을 CLAUDE.md에도 반영합니다.
특히 화면 목록, 데이터 모델, API 구조를 CLAUDE.md에 추가하여
Claude가 코드 작성 시 바로 참고할 수 있게 합니다.

### Step 6: 최종 확인

```
✅ Step 5 완료! 상세 PRD가 작성되었습니다.

📋 docs/prd/ROADMAP.md — 경험 로드맵
📄 docs/prd/{각 경험별 PRD 파일들}

다음 단계: /6-prototype 을 입력해주세요.
```
