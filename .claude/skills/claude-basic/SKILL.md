# Claude Basic

Claude Code의 핵심 개념 6가지를 비개발자도 이해할 수 있게 설명하는 skill입니다.

## When to Use

- 사용자가 `/claude-basic` 을 입력했을 때
- "Claude 기본", "Claude 사용법", "기초 알려줘" 등을 요청할 때

## Instructions

### Step 0: 최신 정보 확인

**출력 전에 반드시** 아래를 수행하세요:

1. WebSearch 도구로 "Claude Code features 2026", "Claude Code MCP tools skills" 등을 검색하여 최신 변경사항이 있는지 확인

**변경이 감지된 경우:**

#### 0. 기본 사용법

```
┌─────────────────────────────────────┐
│  터미널에서 claude 입력              │
│         ↓                           │
│  대화하듯이 요청                     │
│  "로그인 페이지 만들어줘"             │
│         ↓                           │
│  Claude가 코드를 짜고 파일을 만듦     │
│         ↓                           │
│  결과 확인!                          │
└─────────────────────────────────────┘
```

**비유**: 카카오톡처럼 대화하면 돼요. "이거 해줘"라고 말하면 알아서 해줍니다.

**바로 해보기**:
```bash
# 터미널에서 입력
claude

# Claude가 시작되면 이렇게 말해보세요
"현재 폴더에 있는 파일 목록을 알려줘"
```

**자주 쓰는 것들**:
```
/help          도움말
/clear         대화 초기화
/model         AI 모델 변경 (빠르게/똑똑하게)
/compact       대화가 길어지면 요약
Ctrl+C         현재 작업 중단
```

#### 1. MCP (Model Context Protocol)

```
MCP 없는 Claude          MCP 있는 Claude
┌──────────┐            ┌──────────────────┐
│ 🧠 두뇌만 │            │ 🧠 두뇌           │
│          │            │ + 👀 웹을 볼 수 있음│
│ 코드만    │            │ + 🖐️ 디자인 가능   │
│ 짤 수 있음│            │ + 📢 슬랙 전송     │
└──────────┘            │ + 🌐 브라우저 조작  │
                        └──────────────────┘
```

**비유**: 스마트폰에 앱을 설치하는 것처럼, Claude에게 새로운 능력을 추가하는 거예요.
카메라 앱을 깔면 사진을 찍을 수 있듯이, Playwright MCP를 연결하면 웹사이트를 볼 수 있어요.

**바로 해보기**:
```bash
# 현재 연결된 MCP 확인
claude mcp list
```

**예시 — Slack으로 메시지 보내기**:
```
"#general 채널에 '오늘 배포 완료했습니다!' 라고 보내줘"
→ Claude가 Slack MCP를 통해 직접 메시지를 보냅니다
```

#### 2. Skill (슬래시 커맨드)

```
일반 대화                    Skill 사용
┌────────────────────┐     ┌────────────────────────┐
│ "ESLint 설정하고     │     │                        │
│  Prettier 깔고      │     │  /4-ground-rule-setup   │
│  .prettierrc 만들고 │     │                        │
│  ..."              │     │  → 한 줄이면 끝!        │
│  (길고 복잡)        │     │                        │
└────────────────────┘     └────────────────────────┘
```

**비유**: TV 리모컨의 버튼이에요. "채널1" 버튼 하나로 복잡한 주파수 설정이 되듯이,
`/deploy` 한 줄이면 복잡한 배포 과정이 자동으로 진행돼요.

**바로 해보기**:
```
/plugin-guide     → 설치된 플러그인 설명
/help-claude      → 막힌 문제 해결
/wrap-up          → 세션 정리
```

**나만의 skill 만들기**:
```
.claude/skills/나의스킬/SKILL.md 파일을 만들면
/나의스킬 으로 실행할 수 있어요!
```

#### 3. Subagent (부하 직원)

```
혼자 일하는 Claude          Subagent 쓰는 Claude
┌──────────────┐          ┌──────────────────────┐
│ 🤖 Claude     │          │ 🤖 Claude (매니저)     │
│              │          │   ├→ 🤖 A: 파일 검색   │
│ 하나씩       │          │   ├→ 🤖 B: 코드 분석   │
│ 순서대로     │          │   └→ 🤖 C: 테스트 실행  │
│ 처리         │          │                       │
│ (느림)       │          │   동시에 처리! (빠름)   │
└──────────────┘          └──────────────────────┘
```

**비유**: 사장님이 직원 3명에게 각각 다른 일을 시키는 거예요.
"너는 자료 찾아, 너는 분석해, 너는 보고서 써" — 동시에 진행되니까 빨라요.

**실제로 일어나는 상황**:
```
여러분: "이 프로젝트에서 TODO가 있는 파일 찾아서, 각각 해결 방안 알려줘"

Claude 내부:
  → Agent A: src/ 폴더에서 TODO 검색 중...
  → Agent B: test/ 폴더에서 TODO 검색 중...
  → Agent C: docs/ 폴더에서 TODO 검색 중...
  → (결과 종합해서 답변)
```

여러분이 직접 subagent를 호출할 필요는 없어요.
Claude가 복잡한 작업을 받으면 알아서 subagent를 씁니다.

#### 4. Tools (Claude의 도구들)

```
Claude가 쓸 수 있는 도구 상자
┌─────────────────────────────────────────┐
│ 📂 파일 관련                             │
│    Read(읽기) Write(쓰기) Edit(수정)      │
│    Glob(파일찾기) Grep(내용검색)           │
│                                         │
│ 💻 실행                                  │
│    Bash(터미널명령)                       │
│                                         │
│ 🌐 웹                                   │
│    WebSearch(검색) WebFetch(페이지가져오기) │
│                                         │
│ 🙋 사용자 소통                            │
│    AskUserQuestion(질문하기)              │
│                                         │
│ 🤖 작업 관리                              │
│    Agent(부하직원) Skill(스킬호출)         │
│    TaskCreate/Update(할일관리)            │
└─────────────────────────────────────────┘
```

**비유**: 요리사에게 칼, 냄비, 오븐이 있듯이 Claude에게도 도구가 있어요.
"파일 읽어줘"하면 Read 도구를, "검색해줘"하면 WebSearch 도구를 씁니다.

**이걸 왜 알아야 하나요?**
Skill을 만들 때 "AskUserQuestion 도구로 질문하세요"처럼 도구를 지정할 수 있거든요.

**가장 많이 쓰이는 도구 5개**:
| 도구 | 하는 일 | 예시 |
|------|---------|------|
| `Bash` | 터미널 명령 실행 | `bun run dev` 실행 |
| `Read` | 파일 읽기 | 코드 파일 내용 확인 |
| `Edit` | 파일 수정 | 버그 있는 코드 고치기 |
| `AskUserQuestion` | 사용자에게 질문 | "어떤 걸로 할까요?" 선택지 제시 |
| `WebSearch` | 웹 검색 | 최신 라이브러리 정보 찾기 |

#### 5. Teams Mode

```
일반 모드                    Teams 모드
┌──────────────┐           ┌───────────────────────┐
│ 🤖 Claude 1명 │           │ 🤖 Claude A: 프론트    │
│              │           │ 🤖 Claude B: 백엔드    │
│ 모든 걸      │           │ 🤖 Claude C: 테스트    │
│ 혼자 함      │           │                       │
│              │           │ 각자 전문 분야에서      │
│              │           │ 동시에 작업!            │
└──────────────┘           └───────────────────────┘
```

**비유**: 혼자 하는 자영업 vs 직원이 있는 가게.
혼자면 주문도 받고 요리도 하고 서빙도 해야 하지만,
Teams면 각자 역할을 나눠서 동시에 일해요.

**Subagent랑 뭐가 달라요?**
```
Subagent = 한 Claude 안에서 내부적으로 일을 나눔 (자동)
Teams    = 여러 Claude 세션이 실제로 동시에 돌아감 (수동 설정)
```

**사용 예시**:
```
Claude A 터미널: "프론트엔드 로그인 화면 만들어줘"
Claude B 터미널: "백엔드 인증 API 만들어줘"
→ 두 개가 동시에 진행되어 시간 절약!
```

### Step 3: 마무리

모든 설명이 끝나면:

```
🎓 Claude Code 핵심 개념 정리 완료!

요약:
  0. 기본 사용법 → 대화하듯 요청하면 됨
  1. MCP → Claude에게 새 능력 추가 (웹, 슬랙, 디자인...)
  2. Skill → /명령어 한 줄로 복잡한 작업 자동화
  3. Subagent → Claude가 내부적으로 일을 나눠서 빠르게 처리
  4. Tools → Claude가 쓰는 도구 33개 (Read, Write, Bash...)
  5. Teams → 여러 Claude가 동시에 각자 역할 수행

💡 가장 중요한 것:
  그냥 말하면 됩니다!
  "이거 해줘"라고 하면 Claude가 알아서 도구를 골라 씁니다.
  위 개념은 "Claude가 어떻게 일하는지" 이해하기 위한 것이지,
  여러분이 직접 다 조작할 필요는 없어요!
```
