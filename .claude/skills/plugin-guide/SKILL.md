# Plugin Guide

설치된 플러그인과 스킬을 한눈에 소개하는 가이드입니다.

## When to Use

- 사용자가 "어떤 플러그인이 있어?", "설치된 skill 알려줘", "플러그인 가이드" 등을 요청할 때 자동 실행

## Instructions

이 skill은 **실행할 때마다 최신 상태를 동적으로 조회**해야 합니다. 하드코딩된 목록을 출력하지 마세요.

### 실행 단계

**Step 1: 현재 설치된 플러그인 목록 조회**
```bash
claude plugin list
```

**Step 2: 각 플러그인의 README 찾아서 읽기**
- 각 플러그인에 대해 아래 경로들에서 README.md를 탐색:
  - `~/.claude/plugins/cache/<마켓플레이스>/<플러그인명>/*/README.md`
  - `~/.claude/plugins/marketplaces/<마켓플레이스>/plugins/<플러그인명>/README.md`
- README에서 다음 정보를 추출:
  - 플러그인 이름과 출처 (마켓플레이스명으로 판단)
  - 제공하는 슬래시 커맨드 (`/명령어`)
  - 핵심 기능 설명
  - 자동 동작 여부

**Step 3: 비개발자 친화적으로 정리해서 출력**

각 플러그인마다 아래 형식으로 출력:

```
### 플러그인이름 (출처)

| 항목 | 내용 |
|------|------|
| 만든 곳 | 출처 |
| 명령어 | 슬래시 커맨드 또는 "자동 동작" |
| 설명 | 비개발자도 이해할 수 있는 쉬운 설명. 비유나 예시 포함 |
| 언제 쓰나요 | 구체적 사용 시점 |
```

### 출력 규칙

1. **쉬운 말로**: 기술 용어는 괄호 안에 쉬운 비유를 추가 (예: "LSP(코드 맞춤법 검사기)")
2. **커맨드 구분**: 슬래시 커맨드가 있으면 사용 예시도 한 줄 추가, 자동 동작이면 "별도 조작 불필요"로 표기
3. **주의사항**: 동시 사용 시 주의할 플러그인 조합이 있으면 마지막에 안내
4. **관리법 안내**: 마지막에 항상 아래를 출력:
   ```
   - 플러그인 비활성화: `claude plugin disable 플러그인이름`
   - 플러그인 활성화: `claude plugin enable 플러그인이름`
   - 전체 목록 확인: `claude plugin list`
   - 새 플러그인 설치: `claude plugin install 이름@마켓플레이스 --scope project`
   ```
