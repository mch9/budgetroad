# One-Pager PRD

작은 기능이나 내부 도구처럼 상세 기획이 과한 작업에 대해,
기획 의도와 결정 사항을 **한 장짜리 PRD**로 기록하는 워크플로우.

## When to Use

- 사용자가 `/prd-onepager`를 입력했을 때
- 작은 기능, 내부 도구, 기술 기능 등 `/prd-collab` 12단계가 과할 때
- "간단한 기획서", "one-pager", "가볍게 정리"라는 키워드

## 언제 `/prd-collab` 대신 이걸 쓰는가?

```
/prd-collab (12단계)          /prd-onepager (4단계)
─────────────────────         ─────────────────────
사용자 대면 핵심 기능           내부 도구 / 관리자 기능
UX 흐름이 중요한 기능           데이터/기술 설계가 핵심인 기능
와이어프레임이 필요한 기능       화면이 단순하거나 정형화된 기능
처음 만드는 새로운 경험          기존 서비스에 붙이는 작은 기능
```

## Instructions

### Step 1: 맥락 파악

argument로 기능명이 주어지면 바로 Step 2로.
없으면 AskUserQuestion으로 질문:

- question: "어떤 기능을 기획하려고 하세요? 자유롭게 설명해주세요."
- header: "기능 설명"
- options:
  - label: "직접 설명할게요" / description: "아래에 자유롭게 적어주세요"
  - label: "이전 대화에서 이어서" / description: "이번 세션에서 논의한 내용을 기반으로 정리"

"이전 대화에서 이어서"를 선택한 경우, 현재 대화 맥락에서 논의된 기능을 파악하여 자동으로 초안을 구성한다.

### Step 2: 4가지 질문으로 PRD 채우기

다음 5가지를 **순서대로 하나씩** AskUserQuestion + 자유 입력으로 채운다.
이미 대화에서 답이 나온 항목은 "이렇게 정리했는데 맞나요?"로 확인만 받는다.

**Q1. 왜 만드는가? (목적)**
- 이 기능이 없으면 어떤 문제가 있는지
- 누가 쓰는지 (사용자/운영자/개발자)

**Q2. 뭘 보여주거나 해결하는가? (핵심 요구사항)**
- 구체적으로 어떤 데이터/화면/동작이 필요한지
- 핵심 항목 나열 (3~7개)

**Q3. 어떻게 되면 성공인가? (성공 지표)**
- "좋아진다"가 아닌 구체적 숫자나 상태로 정의
- 예: "핵심 KPI 5개를 30초 내 확인 가능", "퍼널 전환율 이탈 구간 식별 가능"

**Q4. 어떻게 만들 건가? (기술 방향)**
- 기술 스택, 데이터 흐름, 주요 설계 결정
- 기존 시스템과의 연동 방식

**Q5. 지금은 안 만드는 것 (제외 범위)**
- 명시적으로 빼는 것과 그 이유
- "나중에"로 미루는 것

### Step 3: One-Pager 작성 및 저장

4가지 답변을 아래 템플릿으로 정리하여 저장한다.

**파일 경로**: `docs/prd/{feature-hierarchy}/{feature-name}-v0.md`
- feature-hierarchy: CLAUDE.md나 ROADMAP.md의 기존 구조를 따름
- 새로운 카테고리면 사용자에게 확인 후 생성

```markdown
# {기능명} — One-Pager PRD

> 작성일: {YYYY-MM-DD}
> 상태: Draft

## 목적

{왜 만드는가 — 문제, 대상 사용자, 기대 효과}

## 핵심 요구사항

{뭘 보여주거나 해결하는가}

- {항목 1}
- {항목 2}
- {항목 3}
- ...

## 성공 지표

{어떻게 되면 성공인가 — 구체적 숫자나 상태}

- {지표 1}
- {지표 2}

## 기술 방향

{어떻게 만들 건가}

- **데이터**: {DB/API 설계 방향}
- **화면**: {UI 구성 방향}
- **연동**: {기존 시스템과의 관계}

## 제외 범위

{지금은 안 만드는 것}

- {제외 1} — {이유}
- {제외 2} — {이유}

## 결정 로그

| 날짜 | 결정 사항 | 이유 |
|------|----------|------|
| {YYYY-MM-DD} | {결정 1} | {근거} |

---
*Generated with `/prd-onepager`*
```

### Step 4: 확인 및 다음 단계 안내

저장 후 출력:

```
✅ One-Pager PRD 작성 완료!

📄 파일: docs/prd/{path}/{name}-v0.md

다음 단계:
  - 바로 구현하려면 → 이 PRD를 기반으로 작업 시작
  - 상세 기획이 필요하면 → /prd-collab {feature-name}
  - 로드맵에 추가하려면 → docs/prd/ROADMAP.md 업데이트
```

ROADMAP.md가 존재하면, 해당 기능을 로드맵에 추가할지 AskUserQuestion으로 확인한다.

## Rules

- **5가지 질문만** — 그 이상 묻지 않는다
- **1페이지 분량** — 스크롤 없이 한눈에 볼 수 있는 길이 유지
- **결정 로그 필수** — 기획 중 내린 결정과 이유를 반드시 기록
- **대화 맥락 활용** — 이미 논의된 내용은 다시 묻지 않고 정리해서 확인만 받기
- **기존 문서 구조 존중** — docs/prd/ 아래 기존 폴더 구조를 따른다
